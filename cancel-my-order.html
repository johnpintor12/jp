Cancel My Order

Order Management

Context

A customer wants to cancel their recent order. Your job is to check if the order is eligible for cancellation and process it if possible, or explain why it cannot be canceled if it has already been fulfilled.

Step 0: Prerequisite Check

Condition:

This automation triggers only if the customer has provided an order number or email, and a specific order has been identified by a preceding automation (like "Where is my order?").

  • If an "identified order" is present: PROCEED to Step 1.
  • If NO "identified order" is present: DO NOT TRIGGER. The system should first run an order identification automation.

Step 1: Check Fulfillment Status and Tags

Decision Logic:

  1. If Fulfillment Status is "Unfulfilled" AND the order does NOT have the "in progress" tag:

    The order is eligible for cancellation. PROCEED to Step 2, Path A.

  2. If Fulfillment Status is "Unfulfilled" BUT the order DOES have the "in progress" tag:

    The order is likely being processed and may not be cancellable. PROCEED to Step 2, Path B.

  3. If Fulfillment Status is "Fulfilled," "Partially Fulfilled," or "Shipped":

    The order is not eligible for cancellation. PROCEED to Step 2, Path C.

Step 2: Communicate with Customer and Perform Action

A Path A: Order is Eligible for Cancellation
  1. Action: Cancel the order in the system.
  2. Respond to Customer: "Hi [Customer Name], Thanks for reaching out. I was able to successfully cancel your order #[Order Number] as requested. You should receive an official cancellation confirmation via email shortly, and the refund will be processed back to your original payment method within 3-5 business days. Please let me know if there's anything else I can help with!"
  3. Internal Action: Mark ticket as 'Solved'.
B Path B: Order is "In Progress"
  1. Respond to Customer: "Hi [Customer Name], Thanks for reaching out. I see your order #[Order Number] is currently marked as 'in progress' by our fulfillment team. This usually means it's being packed and may no longer be cancellable. I have submitted a cancellation request to the team, and they will do their best to stop the shipment. You will receive an email confirmation if the cancellation is successful. If it has already shipped, you'll receive a shipping notification instead, and you can then initiate a return once it arrives."
  2. Internal Action: PROCEED to Step 3 (Escalate).
C Path C: Order is Already Fulfilled/Shipped
  1. Respond to Customer: "Hi [Customer Name], Thanks for reaching out about order #[Order Number]. I've checked its status, and it has already been fulfilled and is on its way to you. Because it has already left our warehouse, I'm unable to cancel it at this stage. However, you are welcome to initiate a return for a full refund once the package arrives. You can find our return policy here: [Link to Returns Page]. I apologize for any inconvenience this may cause."
  2. Internal Action: Mark ticket as 'Solved'.

Step 3: Escalate "In Progress" Cancellation Request

Internal Action:

For orders that were "in progress" (Step 2, Path B), an internal escalation is required.

  • Action: Assign the ticket to the appropriate internal team or agent responsible for liaising with the fulfillment center.
  • Internal Note: Add a note to the ticket: "Customer requested cancellation for order #[Order Number] which is tagged 'in progress'. Please confirm with fulfillment if cancellation is possible and notify the customer of the final outcome."
  • Ticket Status: Leave ticket status as 'Open' for the internal team to handle.
